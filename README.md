# Crypto 5M Simple Toobit Scalper

ربات اسکالپ ۵ دقیقه‌ای برای تحلیل با OKX و اجرای واقعی روی Toobit.

## قوانین اصلی

- تحلیل فقط از OKX انجام می‌شود.
- اجرای واقعی فقط روی Toobit انجام می‌شود.
- پنل ترید، دستورات، مارجین، لوریج، آمار، سیگنال عادی/Real، اسلات‌ها و ثبت نتایج حفظ شده‌اند.
- جهت اصلی اسکالپ از `1H` گرفته می‌شود.
- قدرت جهت با `15M` تأیید می‌شود.
- `4H` فقط فیلتر خطر است؛ اگر شدیداً خلاف معامله باشد سیگنال رد می‌شود، ولی ربات را بی‌دلیل خفه نمی‌کند.
- `5M` فقط ستاپ می‌سازد و دیگر مستقیم ورود نمی‌زند.
- ورود واقعی فقط با `1M Trigger` انجام می‌شود.
- بدون حمایت/مقاومت دستی.
- بدون AI، DCA، Martingale و trailing.
- امتیاز از ۱۰۰؛ حداقل صدور سیگنال `70`.
- RR پیش‌فرض `1.5` و برای ستاپ خیلی قوی پیش‌فرض `1.8` است.
- SL پشت wick/trigger گذاشته می‌شود و برای اسکالپ محدود است.
- حداقل سود خالص بعد از کارمزد باید حداقل مقدار پنل باشد؛ پیش‌فرض `0.01 USDT`.

## ورود جدید: 5M Setup + 1M Trigger

مشکل نسخه‌های قبلی این بود که روی خود کندل 5M ورود می‌زدند. در اسکالپ ۵ دقیقه‌ای همان کندل می‌تواند ته حرکت، شکست فیک یا استاپ‌هانت باشد. نسخه جدید این خطا را حذف می‌کند:

```text
5M = فقط ستاپ
1M = فقط نقطه ورود
```

یعنی اگر 5M شکست یا sweep بدهد، ربات هنوز وارد نمی‌شود. باید 1M نشان دهد که قیمت بعد از تله/ری‌تست واقعاً در جهت معامله برگشته است.

## تشخیص جهت

برای LONG:

```text
1H صعودی باشد
15M صعودی باشد
4H شدیداً نزولی نباشد
```

برای SHORT:

```text
1H نزولی باشد
15M نزولی باشد
4H شدیداً صعودی نباشد
```

جهت با ترکیب قیمت نسبت به EMA50، شیب EMA50، VWAP، RSI و MACD سنجیده می‌شود. 4H فقط وقتی جلوی معامله را می‌گیرد که واقعاً خلاف جهت قوی باشد.

## ستاپ‌های مجاز 5M

ربات فقط دو مدل ستاپ را قبول می‌کند:

### 1) Liquidity Sweep + Reclaim

برای SHORT:

```text
قیمت سقف چند کندل قبلی را می‌زند
ولی بالای آن نمی‌ماند
دوباره زیر محدوده بسته می‌شود
بعد 1M باید برگشت نزولی بدهد
```

برای LONG برعکس:

```text
قیمت کف چند کندل قبلی را می‌زند
ولی پایین آن نمی‌ماند
دوباره بالای محدوده بسته می‌شود
بعد 1M باید برگشت صعودی بدهد
```

این مدل برای گرفتن ورود بعد از استاپ‌هانت است؛ یعنی ربات دیگر ته دامپ شورت نمی‌زند یا سقف پامپ لانگ نمی‌گیرد.

### 2) Breakout Retest

اگر 5M از محدوده فشرده خارج شد، ورود مستقیم ممنوع است. ربات صبر می‌کند 1M سطح شکسته‌شده را تست کند و دوباره در جهت معامله برگردد.

```text
Breakout 5M = فقط آلارم
Retest + Trigger 1M = ورود
```

## قوانین تریگر 1M

ورود فقط وقتی صادر می‌شود که 1M این شرایط را بدهد:

- کندل 1M در جهت معامله باشد.
- بدنه کندل حداقل `45%` کل کندل باشد.
- کندل نزدیک سقف/کف خودش بسته شود.
- حجم 1M کمتر از حداقل مجاز نباشد.
- RSI خسته نباشد.
- MACD Histogram تازه در جهت معامله بهتر شده باشد.
- قیمت در جهت معامله EMA20 و VWAP را پس گرفته باشد.
- ورود از سطح ستاپ خیلی دور نشده باشد.
- حرکت سه کندل 1M قبل از ورود بیش از حد انجام نشده باشد.

## فیلترهای رد سخت

ربات سیگنال صادر نمی‌کند اگر:

- 1H و 15M همسو نباشند.
- 4H شدیداً خلاف معامله باشد.
- 5M حرکت قابل استفاده نداشته باشد.
- نه sweep/reclaim معتبر وجود داشته باشد نه breakout/retest معتبر.
- 1M تریگر ورود ندهد.
- ورود از سطح ستاپ دور شده باشد.
- SL از سقف مجاز اسکالپ بزرگ‌تر شود.
- سود خالص بعد از کارمزد کمتر از حداقل پنل باشد.

## دستور حذف آمار

در تلگرام بنویس:

```text
حذف آمار
```

این دستور آمار شمارشی، خطاها و رویدادهای مانیتورینگ قبلی را صفر می‌کند، ولی سیگنال‌های باز را نگه می‌دارد و سود/ضرر کل و سود/ضرر امروز را حفظ می‌کند.

## گزارش اتو سیگنال و دلیل رد شدن ارزها

دستور تلگرام:

```text
اتو سیگنال
```

این دستور آخرین چرخه اسکن را نشان می‌دهد:

- کل ارزهای لیست
- تعداد ارزهای اسکن‌شده
- تعداد سیگنال صادرشده
- تعداد ارزهای ردشده
- تعداد ارزهایی که به خاطر داشتن سیگنال باز دوباره اسکن نشدند
- تعداد خطاها و کول‌داون‌ها
- دلایل پرتکرار رد شدن
- آخرین ارزهای ردشده با دلیل

## مانیتورینگ و ثبت نتیجه

- لوپ مانیتورینگ جدا از لوپ اسکن اجرا می‌شود.
- هر سیگنال باز با قیمت OKX برای TP/SL چک می‌شود.
- نتیجه در جدول `signals` و `monitor_events` ثبت می‌شود.
- پیام نتیجه روی سیگنال اصلی ریپلای می‌شود.
- اگر ریپلای تلگرام به هر دلیل شکست بخورد، پیام نتیجه به‌صورت پیام عادی هم ارسال می‌شود تا نتیجه گم نشود.
- برای سیگنال Real، اگر Toobit نشان دهد پوزیشن بسته شده، نتیجه واقعی/خروج هم ثبت می‌شود.

## اجرای مستقیم روی VPS

پروژه فایل shell ندارد. اجرای سرویس باید مستقیم به `main.py` وصل باشد:

```ini
[Service]
WorkingDirectory=/root/forex-signal-bot
EnvironmentFile=/etc/forex-signal-bot.env
ExecStart=/usr/bin/python3 -u /root/forex-signal-bot/main.py
Restart=always
RestartSec=10
```

گیت‌پول و ریستارت بدون پاکسازی:

```bash
cd /root/forex-signal-bot || exit
git pull origin main
python3 -m py_compile *.py
sudo systemctl restart forex-signal-bot.service
sudo systemctl status forex-signal-bot.service --no-pager -l
journalctl -u forex-signal-bot.service -n 80 --no-pager
```
