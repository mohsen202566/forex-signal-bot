# ربات تطبیقی کریپتو — OKX / Bybit / Toobit

این پروژه برای Python 3.10 به بالا ساخته شده و یک ربات یکپارچه برای سیگنال‌های **Initial، Medium و Real** است. تحلیل بازار از OKX انجام می‌شود، Bybit منبع جایگزین کامل داده است و Toobit فقط برای اجرای واقعی و خواندن نتیجه مالی استفاده می‌شود.

> تست‌های پروژه هیچ سفارش واقعی ارسال نمی‌کنند. ترید واقعی بعد از هر استارت به‌صورت اجباری خاموش است و فقط با دستور `ترید فعال` روشن می‌شود.

## ساختار

- `main.py`: چرخه عمر و Workerها
- `config.py`: پیش‌فرض‌ها و بازه‌های ثابت
- `models.py`: مدل‌های داده
- `storage.py`: `runtime.db` و `learning.db`، WAL، Backup و Migration
- `symbol_registry.py`: اعتبارسنجی ۱۰۰ نماد مشترک و انتخاب ۳۵ ارز فعال
- `market_data.py`: OKX اصلی و Bybit fallback بدون مخلوط‌کردن داده یک چرخه
- `news_filter.py`: فیلتر اختیاری خبرهای مهم با کش و پنجره قبل/بعد خبر
- `feature_engine.py`: هشت ابزار نرم و پروفایل هفت‌روزه
- بروزرسانی پروفایل: هر بار فقط قدیمی‌ترین پروفایل فعال، بدون صفرشدن Stage/Champion/آمار
- `behavior_engine.py`: رفتار احتمالاتی و مدل ورود
- `tp_sl_engine.py`: Entry/TP/SL تطبیقی و محاسبه هزینه
- `signal_engine.py`: صدور Initial/Medium/Real و قفل کل ارز
- `virtual_monitor.py`: نتیجه مجازی و مسیر بعد از نتیجه
- `scenario_lab.py`: سناریوهای تک‌تغییری
- `learning_engine.py`: ثبت نتیجه، تشخیص Stop و قانون دو Stop واقعی
- `validator.py`: Challenger/Champion، Promotion و Rollback
- `toobit_client.py`: API فیوچرز Toobit
- `trade_engine.py`: ارسال سفارش واقعی بدون انتظار بلاک‌کننده
- `real_monitor.py`: تأیید ۷۰ثانیه‌ای و بررسی نتیجه هر ۶۰ ثانیه
- `command_router.py`: دستورات سریع فارسی
- `telegram_bot.py`: ورودی/خروجی تلگرام
- `telegram_ui.py`: متن پنل‌ها و پیام‌ها
- `self_test.py`: تست‌های آفلاین یکپارچگی

## رفتار شروع

در نصب تازه:

1. ترید واقعی خاموش می‌شود.
2. ۱۰۰ قرارداد مشترک USDT Perpetual در OKX، Bybit و Toobit بررسی می‌شوند.
3. ۳۵ ارز فعال و ۶۵ ارز ذخیره انتخاب می‌شوند.
4. برای هر ۳۵ ارز فعال و هر دو جهت، پروفایل هفت‌روزه ساخته می‌شود.
5. تا تکمیل تمام پروفایل‌ها هیچ سیگنالی صادر نمی‌شود.
6. پیام «ربات آماده شروع است» به تلگرام ارسال می‌شود.
7. Initial و Medium شروع می‌شوند؛ Real همچنان خاموش است.

در Restart یا Git pull، پروفایل‌های سالم دوباره از صفر ساخته نمی‌شوند. دیتابیس‌ها، Championها، نتایج، قفل‌ها و تاریخچه یادگیری حفظ می‌شوند.

## قواعد اصلی

- کل Registry: ۱۰۰ ارز؛ فعال: ۳۵؛ ذخیره: ۶۵.
- هر ارز در کل سطوح و جهت‌ها حداکثر یک سیگنال رسمی باز دارد.
- Initial و Medium محدودیت کلی اسلات ندارند.
- Medium شبیه‌سازی جدی عملکرد واقعی با مارجین، لوریج، کارمزد، اسلیپیج و Funding است.
- همه سیگنال‌ها و سناریوهای جدید از دلار و لوریج فعلی پنل استفاده می‌کنند.
- پوزیشن تا TP یا Stop باز می‌ماند؛ زمان نمایش‌داده‌شده فقط تخمینی است.
- RR شروع ۱٫۵ است، اما TP/SL و RR بعد از اثبات توسط موتور قابل تغییرند.
- حداقل سود خالص Medium/Real برابر ۰٫۰۵ USDT است.
- Real بیش‌ازحد فیلتر نمی‌شود؛ پروفایل ثابت‌شده با شروط اجرایی سالم راه ورود دارد.
- دو Stop واقعی متوالی روی یک ارز، اجرای Real کل همان ارز را فوراً مسدود می‌کند.
- جهت یا جهت‌هایی که در آن دو Stop نقش داشته‌اند به `MEDIUM_RELEARN` می‌روند؛ جهت دیگر نیز تا آزادشدن قفل کل ارز اجازه اجرای Real ندارد.
- بازگشت به Real فقط پس از اصلاح، ساخت Champion جدید، آزمایش Medium و اثبات دوباره سود و وین‌ریت انجام می‌شود.
- یک نتیجه به‌تنهایی Champion را تغییر نمی‌دهد؛ تغییر ابتدا Challenger و سپس روی فرصت‌های آینده تأیید می‌شود.

## دیتابیس‌ها

### `runtime.db`

تنظیمات پنل، سیگنال‌ها، قفل ارز، اسلات، پوزیشن واقعی، Snapshot حساب، Offset تلگرام و سلامت Workerها.

### `learning.db`

نمادها، پروفایل‌ها، نتایج، سناریوها، نسخه‌ها، Champion/Challenger، تشخیص Stop، مسیر بعد از نتیجه و ممیزی یادگیری.

هر دو دیتابیس با SQLite WAL کار می‌کنند. فایل‌های DB در Git قرار نمی‌گیرند. Backup زمان‌بندی‌شده و Backup هنگام خاموش‌شدن امن انجام می‌شود.

## نصب روی VPS

```bash
cd /root
git clone <YOUR_REPOSITORY_URL> forex-signal-bot
cd /root/forex-signal-bot
sudo ./install_service.sh
sudo nano /etc/forex-signal-bot.env
sudo systemctl start forex-signal-bot.service
journalctl -u forex-signal-bot.service -n 100 -f
```

روی VPS باید `python3 --version` حداقل 3.10 باشد.

کلید Toobit باید دسترسی لازم برای Futures Trading و خواندن حساب/پوزیشن داشته باشد. برداشت (Withdraw) برای این ربات لازم نیست و بهتر است برای کلید غیرفعال باشد.

## آپدیت


این اسکریپت قبل از Restart تست آفلاین، `py_compile` و `compileall` را اجرا می‌کند. خود ربات نیز دیتابیس‌ها را حفظ می‌کند و ترید واقعی بعد از Restart دوباره خاموش خواهد بود.

```bash
cd /root/forex-signal-bot
.venv/bin/python -m unittest -v self_test.py
.venv/bin/python -m py_compile ./*.py
```

مجموعه فعلی شامل **۲۸ تست آفلاین** است. تست‌ها شامل حفظ دیتابیس بعد از Restart، بازه دستورات، قفل کل ارز، اسلات اتمیک، محاسبات ابزار/TP/SL، سناریو، Challenger آینده‌نگر، قانون دو Stop واقعی، Payload سفارش Toobit، alias نمادها، fallback یکپارچه، بازیابی معامله‌ای که در مهلت ۷۰ثانیه‌ای باز و بسته شده، جلوگیری از تطبیق نتیجه قدیمی هم‌جهت و رفتار مانیتور واقعی است.

## نکات عملیاتی

- نبود یا خرابی یک نماد باعث Crash کل ربات نمی‌شود؛ نماد Skip/Cooldown می‌شود و اسکن ادامه دارد. Aliasهای رایج هر صرافی و نام‌های رسمی rebrand مانند POL/MATIC، RENDER/RNDR و SONIC/S فقط در خانواده‌های صریح اعتبارسنجی می‌شوند.
- مهلت ۷۰ثانیه‌ای از لحظه شروع واقعی ارسال سفارش محاسبه می‌شود، نه از زمان قرارگرفتن سیگنال در صف.
- Worker سبک تأیید فقط پس از رسیدن Deadline به API درخواست می‌زند؛ Snapshot حساب و نتیجه بسته‌شدن هر ۶۰ ثانیه خوانده می‌شود.
- کش کندل فقط داخل حافظه و به تفکیک `source + symbol + timeframe` است؛ در اسکن‌های بعدی به‌جای دانلود دوباره کل تاریخ، فقط دنباله کوتاه ۱دقیقه‌ای همان منبع دریافت و کندل‌های ۵دقیقه‌ای تازه بازسازی می‌شوند. مقدار تازگی با `ANALYSIS_CANDLE_CACHE_FRESH_SECONDS` تنظیم می‌شود.
- اگر پوزیشن در فاصله ارسال تا تأیید ۷۰ثانیه‌ای باز و بسته شده باشد، قبل از ثبت `FAILED_OPEN` تاریخچه تحقق‌یافته Toobit بررسی می‌شود تا TP/Stop و PnL واقعی از دست نرود.
- خطای API Toobit اسلات را به‌اشتباه آزاد نمی‌کند؛ در چرخه بعدی دوباره بررسی می‌شود.
- پوزیشن‌های باز دستی/خارج از ربات که در Snapshot Toobit دیده شوند نیز اسلات واقعی مصرف می‌کنند و دوبار شمرده نمی‌شوند.
- نبود قطعی Credential پیش از ارسال سفارش، `FAILED_OPEN` ثبت می‌کند تا اسلات بی‌دلیل قفل نماند.
- پیام‌های رد سیگنال پیش‌فرض در Journal چاپ نمی‌شوند؛ با `لاگ رد فعال` روشن می‌شوند.
- Feed خبر اختیاری است. `NEWS_CALENDAR_URL` می‌تواند یک لیست JSON یا آبجکت دارای `events`/`data` باشد؛ زمان رویداد در `timestamp`/`time`/`start` و شدت در `impact`/`importance` قرار می‌گیرد. فقط خبر High/Red/Major یا اهمیت عددی ۳ به بالا مسدود می‌شود. بدون URL هیچ خبر ساختگی تولید نمی‌شود.

## محدودیت مهم پیش از پول واقعی

تست‌های آفلاین صحت منطق داخلی را بررسی می‌کنند، اما تفاوت حساب، مجوز API، قالب پاسخ‌های واقعی و محدودیت منطقه‌ای صرافی فقط روی همان VPS و همان حساب مشخص می‌شود. ابتدا چند روز Medium و سپس اجرای واقعی با مارجین کم انجام شود. Real همچنان فقط با دستور کاربر فعال می‌شود.
