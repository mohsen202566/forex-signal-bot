# ربات UEM یک‌ساعته

## اجرا
```bash
pip install -r requirements.txt
python main.py
```

متغیرهای محیطی لازم:
- `TELEGRAM_BOT_TOKEN`
- `TELEGRAM_CHAT_ID`
- `TOOBIT_API_KEY`
- `TOOBIT_API_SECRET`

## دستورات
- `پنل`، `پنل ترید`، `وضعیت`، `ترید`
- `آمار`، `پنل آمار`
- `سلامت`، `هلس`، `health`
- `ترید فعال` / `ترید خاموش`
- `اتو سیگنال فعال` / `اتو سیگنال خاموش`
- `ترید دلار 20`
- `ترید لوریج 10`
- `حداکثر پوزیشن 3`
- `حذف آمار`
- `ریست سود`

فایل `toobit_client.py` از فایل کاربر بدون تغییر محتوایی کپی شده است.

## کنترل‌های نسخه بازبینی‌شده
- مانیتور سیگنال عادی و واقعی تست واحد شده است.
- سیگنال واقعی تا دریافت نتیجه قطعی از توبیت، اشتباهاً به مجازی تبدیل نمی‌شود.
- بررسی ۷۰ ثانیه‌ای هم پوزیشن باز و هم بسته‌شدن سریع را کنترل می‌کند.
- دستورات فقط از CHAT_ID مجاز پذیرفته می‌شوند و offset تلگرام در دیتابیس حفظ می‌شود.
- دیتابیس‌های نسخه قبلی با ستون‌های جدید به‌صورت خودکار مهاجرت می‌کنند.
- سود روزانه در شروع روز UTC هنگام بازشدن پنل/آمار یا ثبت نتیجه صفر می‌شود.
- فایل toobit_client.py در بازبینی تغییر نکرده است.

## اصلاح واچ تطبیقی و تشخیص پامپ/دامپ
- واچ از سناریوی جهت جدا شده است؛ ابطال LONG یا SHORT باعث خروج ارز از واچ نمی‌شود.
- جهت و سطح هر ارز داخل واچ هر ۱۵ ثانیه بازتحلیل می‌شود.
- فشار خلاف جهت، عبور از ابطال و حرکت معکوس معنادار باعث بازتحلیل فوری می‌شود.
- پامپ و دامپ ۵ دقیقه‌ای با حرکت، پایداری کندل‌ها و افزایش حجم تشخیص داده می‌شود.
- ورود به‌صورت ترتیبی و با حافظه چند Snapshot انجام می‌شود؛ عبور سطح باید حفظ شود و جریان معاملات و Microprice پایدار باشند.
- فایل toobit_client.py بدون تغییر باقی مانده است.
